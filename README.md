# DNSHE 多账号自动续期 Worker

基于 Cloudflare Workers 的 DNSHE 免费域名自动续期工具。支持多账号批量管理、网页面板手动续期、定时自动续期，零服务器成本，小白也能快速部署。

## ✨ 功能特性

- 🔄 **自动续期**：自动检测域名剩余有效期，低于设定阈值自动触发续期
- 👥 **多账号支持**：最多同时管理 20 个 DNSHE 账号，批量处理
- ⏰ **定时执行**：支持自定义 Cron 定时规则，无人值守自动续期
- 🖥️ **可视化面板**：自带网页操作界面，一键手动执行，实时查看日志
- 🛡️ **防限流保护**：内置请求间隔，避免触发平台风控限制
- 💰 **零成本运行**：基于 Cloudflare Workers 免费额度，日常使用完全免费

## 🚀 部署教程

提供两种部署方式，**新手推荐使用方法二**，全程网页操作，无需安装任何软件。

### 方法一：Wrangler CLI 部署（适合有开发基础用户）

1. 安装 Wrangler 命令行工具
```bash
npm install -g wrangler
```

2. 登录你的 Cloudflare 账号
```bash
wrangler login
```

3. 克隆本仓库并执行部署
```bash
git clone <你的仓库地址>
cd dnshe-autorenew-worker
wrangler deploy
```

4. 部署完成后，参考下文「环境变量配置」补充账号信息即可。

---

### 方法二：网页控制台部署（小白推荐 ✅）

全程在浏览器操作，无需下载任何工具，5 分钟即可完成。

#### 第一步：创建 Worker 服务
1. 打开并登录 [Cloudflare 控制台](https://dash.cloudflare.com/)
2. 左侧菜单栏点击 **Workers 和 Pages**
3. 点击右上角 **创建应用程序** → 选择 **创建 Worker**
4. 给服务起个名字（比如 `dnshe-autorenew`），点击 **部署**
5. 部署成功后，点击 **编辑代码** 进入代码编辑页面

#### 第二步：粘贴核心代码
1. 清空编辑器里默认的所有代码
2. 把本仓库 `src/index.js` 里的全部代码完整粘贴进去
3. 点击右上角 **部署** 按钮，等待保存生效

> 这一步已经完成了基础部署，接下来需要配置你的 DNSHE 账号密钥。

#### 第三步：配置账号环境变量
1. 回到 Worker 详情页，顶部切换到 **设置** 标签页
2. 左侧菜单选择 **变量**
3. 在「环境变量」区域，点击 **添加变量**
4. 根据你的账号数量填写变量：

**单账号用户填写：**

| 变量名 | 变量值 | 加密 |
|--------|--------|------|
| `API_KEY` | 你的 DNSHE API Key | ✅ 建议勾选加密 |
| `API_SECRET` | 你的 DNSHE API Secret | ✅ 建议勾选加密 |

**多账号用户填写（序号从 1 开始，最多 20 个）：**

| 变量名 | 变量值 | 加密 |
|--------|--------|------|
| `API_KEY_1` | 第 1 个账号的 API Key | ✅ 加密 |
| `API_SECRET_1` | 第 1 个账号的 API Secret | ✅ 加密 |
| `ACCOUNT_NAME_1` | 第 1 个账号备注名（可选，方便区分） | 不用加密 |
| `API_KEY_2` | 第 2 个账号的 API Key | ✅ 加密 |
| `API_SECRET_2` | 第 2 个账号的 API Secret | ✅ 加密 |
| ... | ... | ... |

5. 填完后点击底部 **保存并部署**

> API Key 和 Secret 可以在 DNSHE 官网个人中心的「API 设置」页面获取。

#### 第四步：设置定时自动执行
默认不会自动运行，需要手动添加定时触发器：
1. 还是在 Worker **设置** 页面，左侧选择 **触发器**
2. 找到 **Cron 触发器** 区域，点击 **添加 Cron 触发器**
3. 在输入框中粘贴以下表达式：
   ```
   0 0 1 */3 *
   ```
4. 下方会自动预览执行时间：**每 3 个月（1、4、7、10 月）的 1 号 UTC 00:00**，对应北京时间当天 08:00
5. 点击 **添加** 保存即可

> ✅ 代码已经内置了定时任务处理逻辑，添加完触发器就会自动生效，无需修改任何代码。

## 🎯 使用说明

### 手动执行续期
1. 打开 Worker 的访问地址（在 Worker 详情页可以找到，格式类似 `xxx.xxx.workers.dev`）
2. 点击页面上的 **开始批量续期** 按钮
3. 页面会实时输出执行日志，可以看到每个域名的处理结果

### 自动执行续期
设置好 Cron 触发器后，系统会按照设定的时间自动运行，无需人工操作。
你可以在 Worker 详情页的 **日志** 中查看每次定时任务的执行记录。

## 🔧 可调参数

如果你想调整运行策略，可以在代码最上方修改以下常量：

| 参数名 | 默认值 | 说明 |
|--------|--------|------|
| `RENEW_BEFORE_DAYS` | `180` | 域名剩余天数 ≤ 该值时，才会触发续期 |
| `LIST_SLEEP_MS` | `300` | 遍历每个域名的间隔时间（毫秒） |
| `RENEW_SLEEP_MS` | `800` | 每次续期请求的间隔时间（毫秒） |
| `FETCH_TIMEOUT` | `10000` | 接口请求超时时间（毫秒） |
| `ACCOUNT_SLEEP_MS` | `1000` | 切换不同账号的间隔时间（毫秒） |

## ❓ 常见问题

**Q：怎么找到我的 Worker 访问地址？**
A：进入 Worker 详情页，页面顶部右侧会显示一个以 `.workers.dev` 结尾的链接，这就是你的工具访问地址。

**Q：可以修改自动执行的时间和频率吗？**
A：可以，在 Worker 设置 → 触发器 → Cron 触发器中修改表达式即可。比如改成每月1号执行用 `0 0 1 * *`，每天执行用 `0 0 * * *`。

**Q：为什么有些域名没有被续期？**
A：只有状态为「已注册」且剩余天数 ≤ 180 天的域名才会自动续期；永久域名、已过期、状态异常的域名会被自动跳过，可以在执行日志里看到具体原因。

**Q：Cloudflare Workers 免费额度够不够用？**
A：完全够用。免费版每天有 10 万次请求额度，本工具每3个月才执行一次，消耗几乎可以忽略不计。

**Q：最多可以添加多少个账号？**
A：代码默认支持最多 20 个账号，按序号 `API_KEY_1` 到 `API_KEY_20` 依次配置即可。

**Q：执行日志显示「接口请求超时」怎么办？**
A：一般是网络波动或 DNSHE 服务器响应慢导致的，稍后手动重试即可。如果频繁出现，可以适当调大代码中的 `FETCH_TIMEOUT` 参数。

**Q：怎么确认定时任务真的运行了？**
A：进入 Worker 详情页 → **日志** → 可以查看历史执行记录，定时任务运行时会输出 `[定时任务]` 开头的运行日志。
